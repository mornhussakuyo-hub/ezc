# EZC 设计文档

`ezc` 全称 `easy copy` ，是一个简单的跨平台终端工具，可以让你高效且方便的在中英混合的目录和文件管理环境中进行文件的复制、粘贴、剪切等操作。配合 `zoxide` 使用体验更佳。第一版仅支持 Arch Linux 和 Windows，之后再扩展其他平台。

## 基础指令：

以下是 `ezc` 第一版预计实现的指令。

### `ezc cp 文件名/目录名`

类似各种图形化系统中的文件资源管理器的复制功能，将指定文件或目录加入一个自我维护的剪切板索引并置于顶端。如果该路径已经存在于索引中，则去重并将原有记录移动到顶端。

`ezc` 维护的剪切板不应真实保存文件，而是保存文件绝对路径。暂时不考虑路径不存在的重新寻找机制，后期再实现。

### `ezc ct 文件名/目录名`

类似各种图形化系统中的文件资源管理器的剪切功能，将指定文件或目录加入一个自我维护的剪切板索引并置于顶端。如果该路径已经存在于索引中，则去重并将原有记录移动到顶端。额外的，这个文件被 `ezc` 锁定；锁定能力分别针对 Arch Linux 和 Windows 实现。如果文件已经被锁定或占用，则提示用户并取消操作。Windows 使用不共享句柄实现强制锁定；Arch Linux 使用 `flock` 排他锁，但受 Linux 文件锁机制限制，主动忽略协作锁的程序仍可访问路径。

### `ezc pst [目标目录]`

粘贴剪切板中顶端的文件或目录。传入目标目录时粘贴到指定目录；不传入时粘贴到当前目录。复制项粘贴成功后保留索引，剪切项粘贴成功后删除索引。目标目录中存在同名文件或目录时，询问用户如何处理。

目录可以正常递归粘贴，但目标目录不能是源目录自身或源目录的子目录。遇到这种情况时提示用户并取消操作。

执行粘贴时检测源路径是否失效。如果路径已经不存在，则提醒用户并自动清理对应索引，不再尝试重新寻找文件。

### `ezc pad`

在TUI界面中浏览剪切板，应当以文件索引的形式逐行展示文件路径或目录路径，通过上下键移动，右方向键进入选择菜单，选择是从剪切板中移除该文件还是粘贴。浏览时检测路径是否失效；如果路径已经不存在，则提醒用户并自动清理对应索引。

### `ezc rm [文件名/目录名]`

从剪切板中移除某个文件或目录的索引。执行移除前需要询问用户确认；未指定文件或目录时，从剪切板列表中选择。如果这个文件是复制的，移除索引后不执行其他操作；如果是剪切的，则先解除锁定再移除索引。



`ezc cp` 和 `ezc ct` 可以不输入文件名或目录名，此时打开一个下拉菜单显示当前目录下的所有文件和目录，通过上下键选择，按 Enter 键确定。可以添加参数 `-h` 或 `--hide` 显示隐藏文件和隐藏目录；如果直接输入隐藏文件名或目录名，则不需要该参数。

剪切板索引需要在不同的 `ezc` 命令进程之间共享，但不做跨系统重启的持久化；系统重启后清空索引。
